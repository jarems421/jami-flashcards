Implement the database schema EXACTLY with Prisma + PostgreSQL.

MODELS (do not rename fields):

Deck
- id (uuid, primary key)
- name (string, required)
- parentDeckId (uuid, nullable, FK -> Deck.id)
- createdAt (datetime)
- updatedAt (datetime)
- settingsOverride (json, nullable)

Note (CONTENT ONLY)
- id (uuid)
- deckId (uuid, FK -> Deck.id)
- noteTypeId (uuid, FK -> NoteType.id)
- fields (json key/value, e.g. {"Front":"...","Back":"..."})
- tags (string array)
- media (json array of {type, filename, url}, default empty array)
- createdAt
- updatedAt

Card (SCHEDULING ONLY — no content fields)
- id (uuid)
- noteId (uuid, FK -> Note.id)
- deckId (uuid, FK -> Deck.id)
- templateId (string)
- state (enum: NEW, LEARNING, REVIEW, RELEARNING)
- dueAt (datetime)
- intervalDays (float)
- easeFactor (float, default 2.5)
- learningStepIndex (int, default 0)
- reps (int, default 0)
- lapses (int, default 0)
- lastReviewedAt (datetime nullable)

ReviewLog (append-only; never update existing rows)
- id (uuid)
- cardId (uuid, FK -> Card.id)
- reviewedAt (datetime)
- rating (enum: AGAIN, HARD, GOOD, EASY)
- responseTimeMs (int)
- previousState (enum)
- newState (enum)
- previousIntervalDays (float)
- newIntervalDays (float)

NoteType
- id (uuid)
- name (string)
- fields (string array ordered)  // e.g. ["Front","Back"]

Template
- id (string, primary key)       // e.g. "basic", "reverse-fwd", "reverse-bwd", "cloze", "image-occlusion"
- noteTypeId (uuid FK -> NoteType.id)
- frontHTML (string)
- backHTML (string)

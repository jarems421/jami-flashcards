Implement the spaced repetition scheduler as a pure function + DB transaction wrapper.

Defaults (read from Deck settings with fallback defaults):
learningStepsMinutes = [1,10]
graduatingIntervalDays=1
easyIntervalDays=4
startingEaseFactor=2.5
easyBonus=1.3
hardMultiplier=1.2
lapseStepsMinutes=[10]
lapseIntervalMultiplier=0.5
minEaseFactor=1.3
maxIntervalDays=36500

State transitions (exact):
NEW:
- On first grade, treat as LEARNING step 0 scheduling (do not leave it NEW after grading)

LEARNING:
- AGAIN: learningStepIndex=0, dueAt = now + learningStepsMinutes[0] minutes
- HARD: keep same step index, dueAt = now + learningStepsMinutes[current] minutes
- GOOD: advance step index; if past last step -> state=REVIEW, intervalDays=graduatingIntervalDays, dueAt=now+intervalDays days
- EASY: state=REVIEW, intervalDays=easyIntervalDays, dueAt=now+intervalDays days

REVIEW:
- AGAIN:
  - state=RELEARNING
  - lapses += 1
  - intervalDays = max(1, intervalDays * lapseIntervalMultiplier)
  - learningStepIndex=0
  - dueAt=now + lapseStepsMinutes[0] minutes
- HARD:
  - intervalDays = max(1, intervalDays * hardMultiplier)
  - easeFactor = max(minEaseFactor, easeFactor - 0.15)
  - dueAt=now + intervalDays days
- GOOD:
  - intervalDays = min(maxIntervalDays, intervalDays * easeFactor)
  - dueAt=now + intervalDays days
- EASY:
  - intervalDays = min(maxIntervalDays, intervalDays * easeFactor * easyBonus)
  - easeFactor = max(minEaseFactor, easeFactor + 0.15)
  - dueAt=now + intervalDays days

RELEARNING:
- Use lapseStepsMinutes like LEARNING:
  - AGAIN: reset stepIndex=0, dueAt=now+ lapseStepsMinutes[0]
  - HARD: repeat step, dueAt=now+ current lapse step minutes
  - GOOD: advance step; if finished steps -> state=REVIEW and dueAt based on stored intervalDays (days)
  - EASY: immediately finish steps -> state=REVIEW and dueAt based on stored intervalDays

Clamps:
- easeFactor >= minEaseFactor
- intervalDays <= maxIntervalDays

Other required updates per grade:
- reps += 1
- lastReviewedAt = now

Testing:
- Add unit tests (vitest or jest) covering each transition and clamping.
- Add tests verifying dueAt increments are correct (minutes vs days).

Prompt: Knowledge Constellations Progress System (for a revision/flashcard app)

Build a “Knowledge Constellations” progress system for my revision/flashcard app.

Core Loop
	•	Every time the user completes a goal (e.g., completes a revision task, finishes a flashcard session, hits a study target), award exactly 1 Star.
	•	Stars are never removed (no penalties). Progress is purely additive.
	•	Stars belong to the user’s current active Constellation.

Constellations (100-Star Chapters)
	•	A Constellation is a named collection of up to 100 Stars.
	•	The current constellation fills as the user earns stars from goals.
	•	When the user reaches 100 Stars, that constellation is marked Complete, and a new empty constellation begins automatically (rinse and repeat).
	•	The user can create multiple constellations over time; each constellation should be saved and viewable permanently.

Star Canvas + Custom Arrangement
	•	Each constellation has an editable “Sky Canvas” where stars are displayed as points of light on a dark background.
	•	When a new star is earned, it should appear on the canvas (initial placement can be a default/random location that doesn’t overlap too heavily).
	•	The user can drag and reposition stars freely to create any shapes/constellations they want (letters, symbols, animals, abstract art, etc.).
	•	The user can organize stars in any configuration; there is no enforced “correct” layout.

Naming
	•	Each constellation can be named by the user.
	•	The name is displayed in the constellation view and in the constellation gallery/list.

Background Feature
	•	The user can select any constellation, even if incomplete, and set it as the app background.
	•	When a constellation is the background, it should still reflect ongoing progress (if it’s the active constellation) and show its current star layout.

Isolated “Gallery” View
	•	Provide a dedicated “Isolated Constellation View” where the constellation appears alone with minimal UI.
	•	The user should be able to zoom and pan to view the constellation clearly.
	•	This view should feel calm and aesthetic (minimal clutter).

Rare Stars (Enhancement)
	•	Add rare milestone stars that are visually distinct but still editable/movable like normal stars.
	•	Suggested rule:
	•	Every 10th star is slightly brighter or subtly different.
	•	Every 25th star is more distinct (e.g., bigger glow or unique sparkle).
	•	The system must track which stars are rare based on the count earned within that constellation.
	•	Rare stars should not create pay-to-win mechanics; they are purely visual.

Gentle Completion Moment (Enhancement)
	•	When the user earns the 100th star in a constellation:
	•	Trigger a gentle, calming completion animation (soft glow / slow shimmer / subtle pulse).
	•	Avoid loud celebrations, fireworks, confetti, or high-intensity effects.
	•	Show a simple message like: “Constellation Complete.”
	•	After completion, automatically start a new active constellation with 0 stars, while keeping the completed one saved and viewable.

Data / State Requirements
	•	Persist the following per constellation:
	•	constellation id
	•	name
	•	star list (each star has: id, position x/y, earned order/index, rarity type)
	•	created date/time (optional but helpful)
	•	completion status (complete when star count hits 100)
	•	Persist global user settings:
	•	which constellation is selected as the app background
	•	which constellation is currently active

UX Requirements
	•	The experience should feel calming, minimal, and motivating.
	•	The user should always feel progress and ownership.
	•	Prioritize simplicity and reliability over complex gamification.

Implement this system end-to-end so it integrates naturally into a revision app: star awarding is triggered by “goal completion” events, and the constellation views are accessible from the main UI.
